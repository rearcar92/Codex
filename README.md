# Codex Growth OS

Codex Growth OS는 현재 업무를 프로젝트 단위로 기록하고, 그 기록을 포트폴리오/업무 자동화/데이터 분석/AI 서비스 아이디어로 재활용하기 위한 개인 지식 운영체계입니다.

이 저장소는 Obsidian Vault이자 Git 저장소입니다. Obsidian에서는 `[[링크]]`와 MOC를 중심으로 지식을 탐색하고, Git에서는 문서와 자동화 산출물의 변경 이력을 관리합니다.

## 시작점

Obsidian에서 가장 먼저 열 문서는 [[Home]]입니다.

가장 중요한 출발점은 아래 두 문서입니다.

- [[MOC - CMDS Flow]]
- [[MOC - 프로젝트]]
- [[MOC - 현재 업무]]

## 운영 방식

### 1. 현재 업무를 기록한다

일단 업무 기록은 [[MOC - 현재 업무]]에서 시작합니다.

업무를 단순 일지로 남기지 않고 아래 질문으로 정리합니다.

- 어떤 문제가 있었나?
- 내가 어떤 판단을 했나?
- 다시 반복될 일인가?
- 포트폴리오 소재가 될 수 있나?
- 자동화하거나 데이터로 볼 수 있나?

### 2. 프로젝트로 묶는다

반복되거나 의미 있는 업무는 [[MOC - 프로젝트]] 아래에 프로젝트로 묶습니다.

예:

- [[MOC - PG 정산 프로젝트]]
- 관리자 화면 개선 프로젝트
- 업무 자동화 프로젝트
- 포트폴리오 리빌드 프로젝트

### 3. 프로젝트에서 자산을 파생시킨다

프로젝트는 아래 자산으로 이어집니다.

```text
프로젝트
→ 업무 기록
→ 도메인 지식
→ 포트폴리오 소재
→ 자동화 아이디어
→ 데이터 분석 질문
→ AI 서비스 아이디어
```

### 4. CMDS 단계는 프로퍼티로 관리한다

각 노트에는 필요하면 Obsidian 프로퍼티로 현재 단계를 표시합니다.

```yaml
단계: 연결
상태: 초안
```

기본 흐름은 아래와 같습니다.

```text
Connect -> Merge -> Develop -> Share
```

- `connect`: 현재 업무, 문제 발견, 아이디어 수집
- `merge`: 도메인 지식, 참고자료, 맥락 통합
- `develop`: 프로젝트화, 자동화, 데이터 분석, AI 서비스 기획
- `share`: 포트폴리오, 면접 답변, 외부 공유 산출물

자세한 기준은 [[MOC - CMDS Flow]]에서 관리합니다.

### 5. 큰 산출물은 Drive에 둔다

PDF, 이미지, 엑셀, 발표자료 등 무거운 파일은 Google Drive에 보관하고, 링크는 [[98_outputs-index/drive-index|Google Drive Index]]에 남깁니다.

## 폴더 역할

- `00_inbox`: 임시 메모와 아이디어
- `01_projects`: 현재 업무와 프로젝트 기록
- `02_portfolio-career`: 포트폴리오, 면접, PM/PO 성장 소재
- `03_data-analysis`: 데이터 분석, SQL, 지표 설계
- `04_ai-service`: 개인 AI 서비스 기획과 MVP
- `05_automation-ideas`: 업무 자동화와 Codex 스킬 아이디어
- `90_references`: 외부 참고자료 요약
- `98_outputs-index`: Drive 산출물 링크 인덱스
- `99_archive`: 완료/보류 자료
- `etc/_assets`: Obsidian 첨부파일 보관
- `etc/_templates`: Obsidian 템플릿 보관

## 핵심 원칙

- 폴더는 보관 위치이고, MOC는 생각의 지도입니다.
- 프로젝트가 중심이고, 도메인 지식은 프로젝트를 설명하는 재료입니다.
- CMDS 단계는 폴더명보다 프로퍼티로 관리합니다.
- 중요한 노트는 반드시 하나 이상의 MOC에 연결합니다.
- 검색보다 먼저 MOC를 열고, 필요한 문서를 링크로 따라갑니다.
- 완성된 답보다 연결 가능한 초안을 먼저 남깁니다.
